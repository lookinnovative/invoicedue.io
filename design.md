## Document Purpose

This document describes the visual layout, screen structure, and interaction patterns for InvoiceDue. It uses ASCII diagrams to communicate design intent without specifying visual styling details.

**References**: This document is derived from `scope.md` and `mvp.md`. All design decisions must align with those documents.

---

## 1. Design Principles

| Principle | Description |
|-----------|-------------|
| **Calendly-level simplicity** | User completes setup in under 10 minutes |
| **Minimalist and calm** | Sparse UI, ample whitespace, no visual clutter |
| **Finance-professional** | Serious, trustworthy aesthetic suitable for finance teams |
| **No technical language** | Never mention AI, agents, bots, or automation |
| **Sensible defaults** | Everything works out of the box with minimal configuration |

## 12. Interaction Patterns

### 12.1 Navigation

| Action | Behavior |
|--------|----------|
| Click nav item | Navigate to screen, highlight active item |
| Click logo | Return to Dashboard |
| Click user menu | Show dropdown: Settings, Sign Out |

### 12.2 Tables

| Action | Behavior |
|--------|----------|
| Click row | Open detail panel (Invoices) or navigate to related item |
| Click column header | Sort by column (toggle asc/desc) |
| Click filter | Show filter dropdown |
| Scroll | Table body scrolls, header stays fixed |

### 12.3 Forms

| Action | Behavior |
|--------|----------|
| Submit valid form | Save and show success message |
| Submit invalid form | Show inline validation errors |
| Change field with error | Clear error when field is corrected |
| Click Cancel | Discard changes, return to previous state |
| Click Reset | Restore default values with confirmation |

### 12.4 Modals

| Action | Behavior |
|--------|----------|
| Click X | Close modal |
| Click outside modal | Close modal |
| Press Escape | Close modal |
| Click primary action | Execute action, close on success |

### 12.5 Alerts

| Action | Behavior |
|--------|----------|
| Dismiss alert | Alert hidden (session only) |
| Warning alerts | Cannot be dismissed |
| Error alerts | Persist until condition resolved |

## 14. Design Exclusions (Authoritative)

The following are explicitly prohibited in the design:

| Exclusion | Rationale |
|-----------|-----------|
| AI/agent/bot terminology | Per scope.md, no technical language |
| Workflow diagrams or builders | Not in MVP per mvp.md |
| Advanced customization controls | Simplicity constraint |
| Multiple themes or customization | Out of scope |
| Animations beyond basic transitions | Keep calm, professional aesthetic |
| Gamification elements | Not appropriate for finance tool |
| Chat interfaces | No conversational UI |
| Real-time updates (WebSocket) | Polling acceptable per scope.md |
